Notes
===================
1. Tests are executed using Browser Stack
2. Test environment can be configured through "test.js" under isentia project root directory, including for both web and mobile applications 
3. Enter "npm test" to start automation scripts running
4. Service credentials "BROSWERSTACK_USERNAME" and "BROSWERSTACK_AUTOMATE_KEY" are configured as system environment variables
5. Screenshots for failures are captured and saved under isentia project root directory as well
6. Please use "sample.png" under isentia project root directory as a reference for command and test result
